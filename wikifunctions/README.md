# WikiLean → Wikifunctions (data pipeline)

The Wikifunctions **verification project** — the Lean proofs (imperative-Python embedding,
`Z13701`/`Z13667` theorems), the spec corpus, the verified composite evaluator, and the
native cross-checks (CPython difftests, lean.py, Dafny, Verus) — has moved to its own repo:

**→ <https://github.com/Deicyde/wikifunctions>**

What remains here is the **data pipeline** that feeds it (and the site's
[/wikifunctions](https://wikilean.jackmccarthy.org/wikifunctions) page): the join of
Wikidata QIDs → Wikifunctions ZIDs → Mathlib declarations, derived from WikiLean's
concept mappings.

## The join

A Wikidata item links to its Wikifunctions function via a `wikifunctionswiki` **sitelink**
(e.g. [Q104752 "coprime integers"](https://www.wikidata.org/wiki/Q104752) ↔
[Z13701 "are coprime"](https://www.wikifunctions.org/wiki/Z13701)). WikiLean already maps
that same QID to a Mathlib declaration, so the QID is the join key connecting an
*executable* Wikifunction to a *machine-checked formal definition*.

## Layout

```
wikifunctions/
├── README.md                       ← this file
├── build_join.py                   ← step 1: measure the addressable set
├── build_specs.py                  ← step 2: enrich with each function's Wikifunctions interface
└── data/
    ├── wikifunctions_universe.jsonl      ← all Wikidata items with a wikifunctions sitelink (QID→ZID)
    ├── wikifunctions_join.jsonl          ← every universe row tagged with WikiLean status
    ├── wikifunctions_join_summary.json   ← headline counts
    ├── wikifunctions_specs.jsonl         ← the addressable functions + their Wikifunctions signatures
    └── wikifunctions_specs_verified.jsonl ← the curated corpus (oracle, faithfulness, tier, Lean spec)
```

Upstream inputs (`concept_layer.jsonl`, `wikidata_universe.jsonl`) live in `../catalog/data/`.
The standalone repo vendors snapshots of these outputs; refresh them there after regenerating.

## Regenerate

```bash
python3 wikifunctions/build_join.py     # → data/wikifunctions_{universe,join,join_summary}
python3 wikifunctions/build_specs.py    # → data/wikifunctions_specs.jsonl
```

Both resolve paths relative to the script and shell out to `curl` (Wikidata SPARQL +
WikiLambda API), so they run from anywhere and avoid the macOS Python CA-bundle issue.
`wikifunctions_specs_verified.jsonl` is a curated artifact (produced with a Mathlib-checked
review pass), not regenerated by these scripts.
